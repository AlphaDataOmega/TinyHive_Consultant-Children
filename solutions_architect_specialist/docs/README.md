## Technical Discovery Framework

### Discovery Types
| Type | Focus |
|------|-------|
| Outcome | Business goals, success metrics |
| Infrastructure | Current tools, integrations |
| Challenge | Pain points, inefficiencies |

### Key Questions
- What outcomes are you trying to achieve?
- What tools are in your current stack?
- Where are your biggest bottlenecks?
- What does success look like?

## Demonstration Best Practices

### Demo Structure
1. **Hook** - Start with customer pain point
2. **Flow** - Show end-to-end workflow
3. **Value** - Connect features to outcomes
4. **Next Steps** - Clear call to action

### Do's and Don'ts
| Do | Don't |
|----|-------|
| Focus on customer use cases | Feature dump |
| Tell a story | Read bullet points |
| Show real workflows | Use contrived examples |
| Connect to business value | Focus only on technical |
