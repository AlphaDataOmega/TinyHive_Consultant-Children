### Product Group Model
| Role | Responsibility |
|------|----------------|
| PM | Product vision, requirements, prioritization |
| EM | Engineering execution, technical decisions |
| Designer | User experience, visual design |
| SET | Quality, test automation |

### RICE Prioritization
| Factor | Question |
|--------|----------|
| **R**each | How many users will this affect? |
| **I**mpact | How much will this move the needle? |
| **C**onfidence | How sure are we of estimates? |
| **E**ffort | How much work is required? |

**Score = (Reach × Impact × Confidence) / Effort**

### Feature Development Lifecycle
1. **Ideation** - Gather ideas from customers, team, market
2. **Validation** - Research, prototype, test assumptions
3. **Build** - Development with iteration
4. **Launch** - NPI process, communication, enablement
5. **Iterate** - Measure, learn, improve

### Design Sprint (5 Days)
| Day | Focus |
|-----|-------|
| 1 | Understand the problem |
| 2 | Diverge - sketch solutions |
| 3 | Decide - choose direction |
| 4 | Prototype - build test version |
| 5 | Test - validate with users |

### Investment Status
- **Investing** - Growing investment
- **Maintaining** - Stable investment
- **Reducing** - Decreasing investment
- **Not Investing** - Minimal/no investment
