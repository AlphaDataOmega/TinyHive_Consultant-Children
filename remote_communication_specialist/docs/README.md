## Async-First Communication

### Core Principles
| Principle | Description |
|-----------|-------------|
| Written first | Start with written communication |
| Public by default | Share openly when possible |
| Document decisions | Capture conclusions in writing |
| Respect timezones | Don't require real-time response |

### When to Use Async
| Use Case | Method |
|----------|--------|
| Status updates | Issue/MR comments |
| Non-urgent questions | Slack channel |
| Decisions with time | Issue discussion |
| Documentation | Merge requests |

## Sync vs Async Decision

### 3-Back-and-Forth Rule
If exchange goes back and forth 3+ times → switch to video call

### When Sync is Appropriate
| Situation | Why Sync |
|-----------|----------|
| Conflict resolution | Tone matters |
| Complex brainstorming | Rapid iteration |
| Relationship building | Personal connection |
| Urgent issues | Time-critical |

## Written Communication

### Guidelines
| Do | Don't |
|----|-------|
| Be concise | Write walls of text |
| Use bullets/structure | Bury key points |
| Assume positive intent | React defensively |
| Provide context | Assume knowledge |

## Video Calls

### Best Practices
| Practice | Reason |
|----------|--------|
| Camera on | Build connection |
| Mute when not speaking | Reduce noise |
| Start on time | Respect schedules |
| End early if possible | Give back time |

## Chat/Slack Etiquette

### Channel vs DM
| Use | Type |
|-----|------|
| Team discussions | Public channel |
| Project work | Project channel |
| Private personnel | DM (sparingly) |
| Sensitive topics | Private channel |

### @Mention Guidelines
| Mention | When to Use |
|---------|-------------|
| @here | Current online, time-sensitive |
| @channel | Everyone in channel |
| @name | Specific person needed |
